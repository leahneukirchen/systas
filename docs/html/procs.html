Procedures

Procedures are those objects which the evaluator knows how to apply to arguments. Built-in functions are procedures as are any functions defined by programs.

A procedure is distinct from the name it is given in any particular context. Thus the symbol `'+' and the primitive procedure bound to the variable called + are distinct objects.

Every procedure has an associated property list (association list). The property procedure-print-name , if it is not #f , is used when displaying or writing a procedure.

Function procedure? 

(procedure? obj)
SCM scm_procedure_p (SCM obj);

Return #t if obj is a procedure, #f otherwise.



Function procedure-properties 

(procedure-properties procedure)
SCM scm_procedure_properties (SCM proc);

Return the property list associated with procedure .



Function set-procedure-properties! 

(set-procedure-properties! procedure list)
SCM scm_set_procedure_property_x (SCM proc, SCM new_val);

Set the property list associated with procedure .



Function procedure-property 

(procedure-property procedure property-name)
SCM scm_procedure_property (SCM proc, SCM property);

Return the named property (or #f ) of procedure .



Function set-procedure-property! 

(set-procedure-property! procedure property-name value)
SCM scm_set_procedure_properties_x (SCM proc,
                                    SCM key,
                                    SCM value);

Set the named property of procedure .



Function closure? 

(closure? obj)

You probably don't want to use this function. Use procedure? instead. This function is for internal use.

Return #t of obj is a closure, #f otherwise.



Function closure-code 

(closure-code obj)

This function is for internal use. It returns the code of a closure.



Function closure-environment 

(closure-environment obj)

This function is for internal use. It returns the environment of a closure.



Function scm_noop 

(noop . args)

Do nothing. Return the first argument or #f .



Function noop2 

(noop2 . args)

Do nothing. Return the second argument or #f .



Function noop3 

(noop3 . args)

Do nothing. Return the third argument or #f .



Function noop4 

(noop4 . args)

Do nothing. Return the fourth argument or #f .



Rationale -- Procedures

procedure? comes from standard Scheme.

procedure-properties and related procedures have no particular rationale -- they might be removed in a future release.

closure? , closure-code , and closure-environment are principally debugging aids. They may or may not have other uses in the future.

The noop family of procedures are surprisingly handy when programming in a functional style.
