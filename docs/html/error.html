Conventions for Errors

up: Systas Scheme
next: Continuations
prev: Catch and Throw

Errors in built-ins usually generate exceptions of type parameter-error . Exceptions are thrown: 

     (throw 'parameter-error msg-string procedure-name . data)

Usually the CAR of data is the errant parameter or a substructure of the errant parameter, but data may be () .

Errors in eval generate exceptions of the type eval-error : 

     (throw 'eval-error procedure-or-procedure-name

Unhandled signals generate an exception of type signal : 

     (throw 'signal signal-name)

A stack depth exceeded condition generates an exception of type stack-depth : 

     (throw 'stack-depth)

Function fatal 

(fatal . args)

Display on the current error port an error message containing the arguments and exit the process with status 1 .



Function error 

(error . args)

Signal a generic error (exception type error ), passing args to the error handler. (Does not return.)
